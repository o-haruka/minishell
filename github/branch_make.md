# branchの作り方

## 現在のブランチの場所を確認

```bash
git status
```

どのブランチから新しいブランチを作りたいか確認する

## ローカルブランチの作成

作成方法は2種類。

- `git branch 作りたいブランチ名`
- `git checkout -b 作りたいブランチ名(✅)`

### git branch 作りたいブランチ名

#### 1. 作るだけ（移動はしない）

「とりあえず後で使うブランチを作っておきたい」という時は、git branch コマンドを使う。

```bash
git bransh 作りたいブランチ名
```

※注意：「ブランチを生やすだけ」で、今いる場所（現在のブランチ）は移動しない。 作業を始めるには、この後 git checkout 作りたいブランチ名 で自力で移動する必要がある。

#### 2. 作って、そのまま移動する（超王道・おすすめ！）

「今からこの機能を作り始めるぞ！」という時は、「作成」と「移動」を同時にやってくれるこちらのコマンドを使う。実務でよく使うらしい。

```bash
git checkout -b 作りたいブランチ名
```

※ -b は「branch」の略。「ブランチを作って、そのままそこにチェックアウト（移動）する」という意味。

## リモートにブランチを認識させる

その通りです！手元（ローカル）で新しくブランチを作っただけでは、GitHub（リモート）には「そんなブランチ知らないよ」という状態になっています。

これをリモート側に認識させて、PR（プルリクエスト）を作れる状態にするための重要なステップが**「初回Push（プッシュ）」**です。

### 🚀 初回Pushのコマンド

新しく作ったブランチ（例：`feature/new-branch`）で作業をして、`git add` と `git commit` まで終わったら、以下のコマンドでGitHubに送り込みます。

```bash
# 「このローカルのブランチを、リモートにも同じ名前で作ってね」という命令
git push -u origin ブランチ名
```

**例:**

```bash
git push -u origin feature/new-branch
```

#### 💡 `-u` って何？（超重要！）

`-u`（または `--set-upstream`）は、**「手元のブランチと、リモートのブランチを紐付ける（追跡させる）」**というおまじないです。

これを最初の1回だけやっておけば、2回目以降コミットを追加した時は、長いコマンドを打たなくても、

```bash
git push
```

または

```bash
git pull
```

と打つだけで、Gitが「あぁ、このブランチはリモートの `feature/new-branch` と紐付いてるんだな」と勝手に判断して通信してくれるようになります。

### 📝 この後の流れ（PR作成まで）

1. **手元でブランチ作成＆移動**
   `git checkout -b feature/〇〇`
2. **コードを書いてコミット**
   `git add .`
   `git commit -m "〇〇機能を追加"`
3. **リモートに初回Push（ここでリモートに認識される！）**
   `git push -u origin feature/〇〇`
4. **GitHubの画面に行く**
   リポジトリを開くと、黄色や緑の帯で **「Compare & pull request」** というボタンが出現しているので、それを押してPRを作成します。
